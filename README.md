# access-log-parser

نحوه اجرای فایل:

با استفاده از دستور زیر فایل `main.py` را اجرا می کنیم:

```
python3 main.py path-to-your-log-file
```

تصمیماتی که در پیاده سازی گرفتم :

1- به جای استفاده از متد هایی مثل `split` از `regex` استفاده کردم تا هم کار آسون تر بشه و هم تمیز تر بشه.

2- برای نگه داشتن لیست آیپی های یکتا از `set` استفاده کردم. این ساختمان داده به صورت خودکار عناصر تکراری رو حذف میکنه و فقط آیپی های یکتا رو نگه میداره.

3- استفاده از `counter` به جای `dictionary`. استفاده از دیکشنری دوتا مشکل داشت :
* اگر یک `endpoint` تازه اومده بود باید یک تکه کد if-else براش میزدم تا به دیکشنری اضافش کنم
* در نهایت برای پیدا کردن دامنه های پرتکرار باید سورت میکردم که اینم کد اضافه میطلبید

پس برای تمیزی و روان شدن کار و کد از counter استفاده کردم

4- طبق سند فایل log رو خط به خط خوندم تا رم پر نشه
5- برای اینکه بعد از یک لاگ خراب برنامه کرش نکنه اومدم از یک بلوک try-except استفاده کردم.
این طوری کار میکنه که بعد از هر خط لاگ خراب یک exception میدیم. اینجا از بخش try میریم داخل بخش exception و تعداد خطوط خراب رو زیاد میکنیم و میریم سراغ ادامه لاگ ها

5- یک مشکلی که به وجود اومد یک بی دقتی ریز بود که برای تعیین نرخ خطا بود که تقسیم بر 0 ممکن بود درست بشه که با یک if-else حلش کردم:
```
if correctLines > 0:
    print(f"Error rate: {error / correctLines * 100:.2f}%")
else:
    print("Error rate: 0.00%")
```
