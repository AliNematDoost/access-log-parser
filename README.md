# access-log-parser

نحوه اجرای فایل:

با استفاده از دستور زیر فایل `main.py` را اجرا می کنیم:

```
python3 main.py path-to-your-log-file --tl threshold-for-401-on-login-page --te threshold-for-5xx-errors --top top-n-endpoints --json if-json-output-needed
```
* ترتیب آرگومان ها اهمیت ندارد
* تمام آرگومان ها به جز مسیر فایل لاگ اختیاری هستند و مقدار پیشفرض دارند:
--tl مقدار پیش فرض 50 دارد

--te مقدار پیشفرض 500 دارد

--top مقدار پیشفرض 10 دارد

--json هم به عنوان فلگ استفاده می شود یعنی اگر وارد نشود خروجی text چاپ میشود

نمونه یک دستور صحیح:
```
python3 main.py /mnt/c/University/Hamravesh/access.log.gz --tl 500 --te 2000 --json --top 1
```

تصمیماتی که در پیاده سازی گرفتم :

1- به جای استفاده از متد هایی مثل `split` از `regex` استفاده کردم تا هم کار آسون تر بشه و هم تمیز تر بشه.

2- برای نگه داشتن لیست آیپی های یکتا از `set` استفاده کردم. این ساختمان داده به صورت خودکار عناصر تکراری رو حذف میکنه و فقط آیپی های یکتا رو نگه میداره.

3- استفاده از `counter` به جای `dictionary`. استفاده از دیکشنری دوتا مشکل داشت :
* اگر یک `endpoint` تازه اومده بود باید یک تکه کد if-else براش میزدم تا به دیکشنری اضافش کنم
* در نهایت برای پیدا کردن دامنه های پرتکرار باید سورت میکردم که اینم کد اضافه میطلبید

پس برای تمیزی و روان شدن کار و کد از counter استفاده کردم

4- طبق سند فایل log رو خط به خط خوندم تا رم پر نشه

5- برای اینکه بعد از یک لاگ خراب برنامه کرش نکنه اومدم از یک بلوک try-except استفاده کردم.
این طوری کار میکنه که بعد از هر خط لاگ خراب یک exception میدیم. اینجا از بخش try میریم داخل بخش exception و تعداد خطوط خراب رو زیاد میکنیم و میریم سراغ ادامه لاگ ها

5- یک مشکلی که به وجود اومد یک بی دقتی ریز بود که برای تعیین نرخ خطا بود که تقسیم بر 0 ممکن بود درست بشه که با یک if-else حلش کردم:
```
if correctLines > 0:
    print(f"Error rate: {error / correctLines * 100:.2f}%")
```

6- برای مدیریت بهتر آرگومان های مختلف در دستور ورودی به جای `sys` از `argparse` استفاده کردم. این کتابخونه به صورت خودکار آرگومان های ورودی رو هندل میکنه و دیگه نیازی نیست کلی کد شلوغ بزنیم تا آرگومان ها رو به ترتیب پردازش کنیم و مشکلاتی مثل نبود یک آرگومان یا تغییر ترتیب آرگومان ها به راحتی حل میشه

7- خیلی جا ها مثل `internalError` به جای استفاده از دیکشنری از `Counter` استفاده کردم. چرا؟ چون اضافه کردن کلیدی که نیست داخل دیکشنری راحت تر میشه. مثلا کد زیر که برای دیکشنری عادیه:
```
if hour in internalError:
    internalError[hour] += 1
else:
    internalError[hour] = 1
```
تبدیل شد به :
```
internalError[hour] += 1
```
کار تمیز تر و سریع تر شد.

8- تصمیم گرفتم برای جلوگیری از کد دوباره و تکراری در هر دو بخش json و text مقدار جهش های ارور 5xx رو قبل این بلاک محاسبه کنم و در هر دو به صورت یکسان استفاده کنم:
```
spikes = dict()
for hour, count in internalError.items():
    if count > tError:
        spikes[hour] = count
```

و از دیکشنری `spikes` در هر دو بلوک json , text استفاده کردم.

9- از hardcode کردن تا جای ممکن پرهیز کردم. مثلا برای مرز تشخیص مهاجم بودن کاربر با ارور های 401 در صفحه login ابتدا مقدار tl را هاردکد کرده بودم روی 50. اما بعد گفتم بهتر است برای این مورد یک آرگومان ورودی تعیین کنم تا کاربر خودش برای این مقدار تصمیم بگیرد. مشابه همین مورد را برای te و top انجام دادم

10- یکی از مشکلات خروجی ناخوانا بودن بود. سعی کردم با پرینت کردن `-` در خروجی کمی بین اطلاعات مختلف فاصله بیندازم تا خواندن آنها برای کاربر گیج کننده نباشد

11- برای خواندن فایل های `.gz` از کتابخانه `gzip` استفاده کردم. شیوه پیاده سازی هم به این صورت هست که پسوند مسیر فایل در path را چک میکنیم اگر `.gz` بود از تابع `gzip.open` برای خواندن فایل لاگ استفاده میکنیم در غیر این صورت از تابع پیشفرض `open` خود پایتون استفاده میکنیم.

12- برای اضافه کردن فیلتر زمانی یه سری مشکلات و البته بهبود هایی اتفاق افتاد که به شرح زیر است:
* اول خواستم برای مقایسه زمان حال با زمان فیلتر صرفا if بذارم اما دیدم برای این کار باید ۳ تا if جدا بذارم. برای همین تصمیم گرفتم برای تمیز تر شدن کد بیام مجموع زمان ها رو به ثانیه حساب کنم و فقط یک if بذارم :
```
timeParts = time.split(':')
hour = timeParts[1]
minute = timeParts[2]
second = timeParts[3].split()[0]

currentTimeInSec = int(second) + int(minute) * 60 + int(hour) * 3600
if currentTimeInSec >= endInSec:
    break
```

البته `endInSec` رو بعد از پارس کردن ارگومان ها محاسبه میکنیم و به تابع پارسر پاس میدیم.
با این کار دیگه بار پردازشی محاسبه زمان پایان به ثانیه توی هر لوپ تکرار نمیشه و فقط یک بار انجام میشه (بهبود پردازشی)

13- بعد از اینکه زمان لاگ ورودی از زمان فیلتر رد شد دیگه لاگ های بعدی رو نمیخونیم و `break` میکنیم و از لوپ خارج میشیم.

14- خط `totalRequests += 1` رو بردم بعد از if چک کردن فیلتر زمان تا درخواستی که زمانش بعد از فیلتر هست شمرده نشه.
حالا چون if فیلتر درخواست خراب رو رد کردیم باید این خط رو توی بلوک `except` هم اضافه کنم تا درخواست های خراب هم شمرده بشن توی تعداد کل درخواست ها.
